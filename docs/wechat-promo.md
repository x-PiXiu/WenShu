# 开源 RAG 知识库：让 AI 只回答「你的」文档

## 不再是一本正经地胡说八道，每个回答都有据可查

---

> 你有没有这样的经历？
>
> 硬盘里躺着几百篇论文、几十份技术文档、无数学习笔记——明明存过，用的时候就是找不到。传统搜索只能按文件名、关键词匹配，问它一个「为什么」，它只能返回一堆关键词高亮的片段。
>
> 如果你的文档能像人一样被提问，直接给你答案呢？

---

**文枢 · 藏书阁**，就是一个干这件事的开源项目。

把文档丢进去，它会自动解析、分块、向量化。然后你就可以用自然语言跟你的文档「对话」了——不是搜索，是真正的问答。

## 它能做什么？

### 和文档对话

上传 PDF、Word、Markdown、纯文本，系统自动完成索引。提问后，AI 会从你的文档中找到最相关的内容，组织成通顺的回答，并附上来源引用。

不是简单的关键词匹配，而是**语义理解 + 关键词检索**的双重召回，再通过 RRF 算法融合排序——问得模糊也能找到对的内容。

<!-- 截图：knowledge-list.png — 知识库文件列表，显示文档类型和分块数 -->

### 闪卡学习

选一份文档，AI 自动生成闪卡。不是简单的填空题，而是有深度的问答卡——先给答案，再解释为什么。内置 SM-2 间隔重复算法，学过的卡片会在最佳时间点提醒复习。

<!-- 截图：flashcard-study-flip.png — 翻卡学习界面 -->

### 博客 + 智能问答

内置博客系统，写完文章发布后，读者可以直接对文章内容提问。相当于每篇博客都自带一个「AI 问答助手」。

<!-- 截图：blog-chat.png — 博客文章右侧智能问答面板 -->

### 自定义 AI 智能体

不满足于默认的问答助手？可以创建多个 AI 人格——一个专门做翻译，一个专门做技术分析，一个专门做知识对比。每个智能体拥有独立的提示词和工具集。

### 全局 Prompt 管理

所有系统提示词（核心功能、工具调用、智能体人设）统一管理，在后台界面直接编辑，保存后立即生效，无需重启。

## 技术上的「讲究」

如果你关心技术细节，这几个点值得关注：

**混合检索引擎**

单纯向量检索擅长「意思相近」但容易漏掉精确关键词；单纯关键词检索擅长「字面匹配」但理解不了语义。文枢两者都用，再通过 RRF 融合——1+1>2。

**模型随时换**

在设置界面切换 LLM 和 Embedding 模型，保存即生效。本地用 Ollama 跑 Qwen，线上切 MiniMax 或 OpenAI，零停机。

<!-- 截图：settings-llm.png — 模型配置界面 -->

**LLM 调用全透明**

内置监控面板，每次 LLM 调用的 Token 消耗、延迟、模型、完成原因都记录在案。做评估的时候，还能跑 Recall@K、MRR 这些专业指标。

<!-- 截图：admin-llm-monitor.png — LLM 监控面板 -->

**A2A + MCP 协议**

实现了 Agent-to-Agent 和 Model Context Protocol，可以与其他 AI Agent 节点协作。现在用不用是一回事，但架构上已经准备好了。

## 部署有多简单？

**下载 → 解压 → 双击启动**，三步搞定。

```bash
# 解压后
./start.sh        # Linux / macOS
start.bat         # Windows 双击即可
```

唯一的前置要求是 JDK 17+。不需要装 Maven、不需要装 Node.js、不需要配 Docker。

首次启动会自动创建配置文件，填入你的 LLM API Key 就能用了。支持 Ollama 本地模型、MiniMax、OpenAI 以及任何 OpenAI 兼容的 API。
